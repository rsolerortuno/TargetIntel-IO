# TargetIntel-IO v0.2.0 — Common Evidence Layer

Issue 208 completed with 183 passing tests; the final release may contain additional tests.

This release adds typed immutable evidence contracts, intrinsic and semantic validation, provenance and lineage, DuckDB operational storage, deterministic Parquet snapshots, and retrieval-attempt auditing. It includes Europe PMC retrieval, deterministic mock extraction, literal quotation verification, documented computed-evidence support, evidence-family construction, exact-duplicate handling, and post-ranking Markdown and HTML evidence cards.

Read-only reporting introduced here opens DuckDB read-only, validates schema without mutations, rejects all write operations, and fails clearly when an explicitly requested store is missing, a directory, malformed, or incompatible. Strict deterministic-isolation tests verify report decoration cannot alter baseline roles, scores, rankings, benchmark results, or sensitivity outputs. The release also includes an executable offline demonstration built entirely from fabricated mock data.

Computed-support verification confirms the presence and structural validity of auditable support metadata; it does not independently reproduce the underlying computation.

v0.2.0 is infrastructure and report decoration. It is not clinical validation, does not qualify biomarkers, and does not validate therapeutic targets. Production LLM extraction, DepMap, single-cell/spatial analysis, patient-response modelling, de novo discovery, and multitumor support remain future work.
