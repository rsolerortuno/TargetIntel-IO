# v0.4.0 Open Targets feasibility profiles

Issue 403 is a pure, offline bridge from Issue 402's immutable Open Targets
fetch result to the Issue 401 `TargetFeasibilityProfile`. It accepts exactly a
`TargetFeasibilityRequest` and an `OpenTargetsFetchResult`; it accepts neither
raw GraphQL, transport/cache objects, paths, scores, ranks, feature tables nor
LLM output. It makes no network, cache, file, scoring, ranking, role,
feature, evidence, LLM, or modality-evaluator call.

`OpenTargetsNormalizationManifest` is a code-owned immutable allowlist. Its
identity includes schema/version, supported record version, source release,
dimension/modality/factor mapping versions, and allowed-source-field hash.
Only Issue 402 `knownDrugs`, `tractability`, and `safetyLiabilities` are read.
Unknown or malformed shapes fail closed or remain explicit limitations. The
profile contains no score, rank, confidence, recommendation, go/no-go, or
clinical conclusion.

Target matching is exact. A request is matched to one resolution with the same
identifier and identifier type, then to one record with the resolved Ensembl
ID (and resolved approved symbol for symbols). Ambiguous, unresolved, invalid,
retrieval-failed, missing-record, and release-mismatch outcomes produce a
terminal result without a fabricated profile. No alias, synonym, fuzzy match,
case repair beyond Issue 402 canonicalization, association rank, or baseline
score is used.

Clinical precedence is descriptive `knownDrugs` source data only: phase,
status, mechanism, drug and disease identifiers remain facts, not target
validation or universal approval. Tractability records retain their source
label/modality/value and map only antibody, small molecule, PROTAC, and other
clinical controlled modalities. PROTAC is normalized only from an explicit
PROTAC record; it is never inferred from small-molecule data. There is no
modality recommendation. Issue 402 retains no doability field, so doability is
explicitly unavailable rather than inferred from other dimensions.

Safety records retain compact source-linked liability fields. An empty retrieved
`safetyLiabilities` collection is a scoped `not_observed` observation; it is
never a safety conclusion. A missing field is `not_available`, while target or
transport failure remains its terminal failure state. Contradictory observations
remain separate, are never averaged or selected, and are visible in coverage.

Every requested dimension and requested tractability modality receives an
immutable `FeasibilityDimensionCoverage`, with stable integer counts and a
descriptive state. Coverage is explicitly not scientific confidence, negative
evidence, a score, or a ranking weight. Build and batch results have canonical
identities over request/fetch/normalization/resolution/record/profile,
coverage, limitations, and sanitized errors; timestamps and local operational
details are excluded. Source release verification propagates unchanged, and an
unverified release is labeled as a limitation.

`targetintel.modality.py` remains the sole modality-evaluation surface. Issue
404 may integrate these profiles explicitly. This research-only layer does not
provide clinical advice, validate targets, or establish safety or causality.
