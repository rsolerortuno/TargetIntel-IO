# v0.3.0 specification — Grounded Literature Copilot

## Objective

v0.3.0 will provide an operational evidence CLI and provider-agnostic LLM integration for source-grounded literature processing. It is optional infrastructure: it must not alter deterministic scores, roles, rankings, benchmark results, sensitivity outputs, or ranking weights. Human review is required before scientific publication.

## Grounding and evidence contract

Extraction may use only retrieved source text or versioned structured source content. Models must never generate PMIDs, DOIs, citations, identifiers, or numerical results. Each exposed literature record requires its exact quotation, stable document identifier, and document location. Computed records require auditable support metadata.

The CLI will record query construction, query expansion, relevance ranking, pagination, retrieved sections, source release, failures, retries, timeouts, and non-execution. It will process documents section-aware where available. Failed and rejected records must be visible in audit reports rather than disappearing.

Before normal report exposure, candidate records undergo deterministic schema and semantic validation plus citation verification. Citation verification, source verification, computation verification, and scientific review are distinct future concepts: computed-support checks establish auditable metadata presence and structural validity, not independent reproduction of a computation.

## Provider-neutral interfaces and provenance

Providers implement a neutral extraction interface over supplied source text. Every run records prompt identity/version, provider, model, model version, extraction configuration, timing, and result status. Offline fake-provider tests must cover success, malformed output, retryable failure, timeout, and non-execution without network access.

Responsibilities are separate:

- The extractor emits structured staging candidates grounded in supplied text.
- The critic checks grounding, omissions, contradictions, limitations, and validation failures; it does not rewrite evidence.
- The writer creates target-level synthesis only from finalized, verified `EvidenceItem` records.

Target synthesis must visibly distinguish supportive, contradictory, limiting, and missing evidence. It must not convert absence into negative evidence or association into causation.

## Quality and review

The release requires a manually reviewed extraction-quality evaluation with explicit sampling, error categories, and citation-grounding measurements. Deterministic audit reports show rejected records and their reasons. Scientific claims intended for publication require human review after these controls.

## Suggested issue decomposition

1. Operational evidence CLI.
2. Provider-neutral model interface and prompt provenance.
3. Real structured extraction.
4. Query expansion and relevance ranking.
5. Full-text and section-aware retrieval.
6. Manually reviewed extraction benchmark.
7. Audit report and scientific critic.
8. Grounded target-level writer.

## Explicitly out of scope

v0.3.0 excludes target-feasibility scoring; DepMap/CRISPR numerical integration; single-cell/spatial numerical analysis; patient-response prediction; automatic ranking-weight changes; graph neural networks; clinical recommendations; and multitumor production support.

It also does not retroactively implement production retrieval-to-finalization behavior from v0.2.0, expanded Europe PMC synonym/full-text retrieval, new evidence-validation vocabularies or schema migrations, a production audit-report CLI, real LLM deployment, or external scientific or clinical validation without separately approved work.
