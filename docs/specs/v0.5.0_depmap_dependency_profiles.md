# v0.5.0 DepMap dependency profiles

Issue 503 is a descriptive, offline layer built on the Issue 501 release
contracts and the Issue 502 local-ingestion artifacts.  It consumes an explicit
`ingestion_snapshot.json`, indexes, and subset matrices; it does not read raw
DepMap files, retrieve data, or create a second parser, gene resolver, model
mapper, release manifest, or ingestion snapshot.

## Boundary

Profiles describe continuous gene-effect and dependency-probability evidence
in a requested structured model context, its non-context models, and the
available pan-cancer model universe.  They do not change scoring, roles,
rankings, selection, modality, cards, reports, benchmarks, or any other
deterministic baseline component.  They make no clinical, causal, safety,
therapeutic, biomarker, or go/no-go claim.  Thresholds are analytical
conventions, not biological truth.

## Context and policy contracts

`DepMapModelContextDefinition` is immutable and versioned.  Its identity
includes the context name and version, metadata-mapping version, exact accepted
and exclusion values, explicit model inclusions/exclusions, minimum counts,
and limitations.  It uses only exact values in structured metadata exposed in
the Issue 502 `model_index.tsv`; it has no substring, regular-expression,
fuzzy, cell-line-name, expression, or dependency-value classification.
Exclusions override inclusion rules.  Missing metadata, metadata-unavailable,
ambiguous, and unreconciled models remain in `model_context_index.tsv`.

`FunctionalDependencyProfilePolicy` fixes measured-model and lineage minimums,
probability thresholds, linear quantiles, missing-value treatment, lineage
position method, contradiction rules, and limitations.  Its deterministic ID
changes when any scientific convention changes.  Gene effect stays continuous;
there is no binary gene-effect dependency call.

## Coverage and summaries

Profiles retain target resolution and matrix coverage separately from model
coverage.  They distinguish unresolved targets, one-matrix-only targets, no
context models, all-missing context values, insufficient context/reference
measurements, and sufficient partial/complete coverage.  Missing values are
excluded from numeric summaries and reported, never converted to zero.

For each context, non-context, and pan-cancer group, summaries retain totals,
measured and missing counts/fraction, mean, median, extrema, quartiles, and
IQR. Probability summaries additionally retain numerator, denominator, and
fraction at every policy threshold.  Contrasts retain raw summaries and report
context minus non-context median and mean values, plus threshold-fraction
deltas for dependency probability.  Parallel context-minus-pan-cancer median,
mean, and threshold-fraction contrasts are retained when their respective
coverage requirements are met.  For gene effect, a negative difference is a
stronger model-dependency signal in the context group.  No p-values or
significance statements are generated.

Lineages use exact `OncotreeLineage` values from the normalized metadata.
Small lineages remain listed as ineligible.  The optional empirical position is
a descriptive percentile of more-negative lineage median gene effect: 100
means stronger than most eligible lineages.  It is neither a probability,
significance result, nor target score.  Context model observations preserve
model identities, range/IQR inputs, and cross-source discordance rather than
averaging contradictions away.

Optional common-essential and pan-dependency roles are represented separately.
The canonical Issue 502 ingestion surface exposes their exact parsed source
labels in `reference_index.tsv`, with its own identity in the ingestion
snapshot. Profiles therefore distinguish explicitly present, explicitly absent
from a validated reference, target unresolved in reference, validation failed,
and unavailable. An unavailable source or absence from a valid reference is
never interpreted as non-essentiality or safety.

## Cross-source observations

Cross-source observations remain separate descriptive records; they are never
averaged into a score.  The profile records configured strong-negative
gene-effect/low-probability and weak-gene-effect/high-probability model-level
discordance, plus: a context signal available in only one matrix; unavailable
paired context evidence for concordance assessment; opposite context-versus-
non-context and context-versus-pan-cancer median directions when observed; and
validated optional-reference membership alongside a more-negative
context-versus-non-context gene-effect median.  These records retain their
source values, availability counts, and reference status.  They are
descriptive limitations, not causal, clinical, safety, or selectivity claims.

## Identity and artifacts

Profile identity includes target identity, release-manifest, ingestion-snapshot
and target-universe identities, plus context and policy IDs.  It excludes
paths, timestamps, users, hosts, and runtime values.  Run identity is likewise
order independent.  Before profiling, the canonical reader recomputes and
checks the declared model-, gene-, and optional reference-index identities and
the SHA-256 identities of both subset matrices against the ingestion snapshot.
Any mismatch fails closed; raw matrices are never used as a fallback.  The
example writes canonical JSON and stable ordering:

- `dependency_profile_manifest.json`
- `dependency_profiles.jsonl`
- `dependency_profile_summary.tsv`
- `dependency_coverage_summary.json`
- `model_context_index.tsv`

The synthetic fixtures are operational tests only and support no biological
finding. Issue 504 will freeze benchmark/discovery universes and leakage rules.
Issue 505 will assess coverage and any possible value for prioritization;
Issue 503 itself deliberately does not benchmark or rank.
