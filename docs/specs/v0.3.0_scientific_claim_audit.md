# v0.3.0 deterministic scientific claim audit

This component consumes Issue 304 `GroundedExtractionResult` staging candidates
and supplied source text. It is a pure, deterministic, research-only wording
and scope audit. It performs no provider execution, retrieval, network access,
filesystem writes, scoring, ranking, citation verification, clinical decision
support, or EvidenceItem construction/promotion.

## Taxonomy and decisions

Taxonomy version `scientific-claim-audit-v1` has immutable rule-to-severity
mapping. Blockers are grounding/source identity failures, therapeutic
recommendations, clinical guidance, preclinical-as-clinical wording, unanchored
identifiers, candidate coverage failures, and invalid input. Warnings cover
association-as-causation, certainty inflation, scope generalization, omitted
explicit limitations, ambiguous support relations, and within-source
contradictions. Unknown rules or severities fail closed. A card is `block` for
any blocker, `review` for warnings without blockers, otherwise `pass`.

Cards retain exact supplied quote, offsets, candidate provenance, deterministic
findings, decision, and explicit `research_only`, `not_clinical_guidance`,
`not_evidence_item`, and human-review-before-promotion declarations. They do
not duplicate the complete source document or retain hidden reasoning. Card and
result identifiers hash canonical public content only, excluding timing,
paths, run identifiers, token use, and provider operational metadata.

## Checks and limitations

Grounding is rechecked case- and whitespace-sensitively: document ID/hash,
integer bounded offsets, and exact `source[start:end] == quote`. The lexical
critic detects recommendation and clinical-guidance wording with documented
conservative negation phrases; causal claim language over associative quotes;
strong certainty over quote uncertainty; explicit preclinical metadata paired
with clinical-benefit wording; broader detectable claim context; omitted
explicit limitations; and identifiers not exactly present in source or quote.
It never expands synonyms, repairs text, uses ontologies, or decides scientific
truth. Opposed `supports`/`contradicts` candidates are flagged only with shared
explicit target and disease anchors; neither is preferred.

`no_candidates` is an audit state, not negative biological evidence. Failed or
inconsistent extraction inputs are invalid/fail-closed. These deterministic
lexical rules have limited language coverage and are an advisory release gate;
human scientific review remains required before any future EvidenceItem
promotion.
